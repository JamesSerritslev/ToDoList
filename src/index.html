README

Built in one day — mostly from a single well-thought-out Cursor prompt — by Jame Serritslev, with follow-up passes on the interactive tour and polish.

Downloads

  • Windows — Projects-Portable.exe (portable) or NSIS installer via npm run build:win:installer
  • macOS — .dmg and .zip (Intel + Apple Silicon)
  • Linux — .AppImage and .deb (x64)
  • GitHub Actions builds all three platforms; tag v* to publish release artifacts

Tech stack

  • Electron 35 — frameless desktop shell (Windows, macOS, Linux)
  • Vanilla JS / HTML / CSS — no React or bundler in the renderer
  • better-sqlite3 — local SQLite with WAL journaling
  • electron-builder — cross-platform packaging

Architecture

  • electron/main.js — BrowserWindow, IPC handlers, DB init, single-instance lock
  • electron/preload.js — contextBridge exposes window.api; renderer has no Node access
  • src/app.js — projects, tasks, notes, drag reorder, keyboard-first input
  • src/guide-tour.js — hands-on tutorial with spotlight overlays and step locks
  • electron/db.js — schema for projects, tasks (is_done + position), and project notes

Data & security

  • Database: tasks.db in the app data folder (%APPDATA%\Projects on Windows, ~/Library/Application Support/Projects on macOS, ~/.config/Projects on Linux)
  • Strict CSP (default-src 'self'), context isolation, no outbound network calls

Development

  • npm start — run locally
  • npm run build — build for the current OS
  • npm run build:win / build:mac / build:linux — platform-specific builds
  • project-paths.json — maps project names to folders for Open in Cursor
  • scripts/ — icon generation, DB migration, seed utilities
